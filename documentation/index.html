Setup

Headcount

Headcounts are recorded as a part of Rock's metric database to ensure forward compatibility and to take advantage of the native data structure. To do this we tie your check-in group structure to a headcount metric structure which is populated once a day by a Rock Job.

Configuring the Job

To store the metrics you first need to create a metric category to store them in. Go to Tools > Reporting > Metrics and add a category with an appropiate name.

Then visit Admin Tools > System Settings > Job Administration and create a new job. Give the job an appropiate title, such as 'Headcount Check-In Sync' and a cron expression that'll ensure it's synced with your check-in structure before you put in any headcount figures. If you're likely to put in figures more than once a week then 0 0 2 1/1 * ? * is a good fit, else if you're likely to record your headcounts only once a week then 0 0 1 ? * MON * will probably work.
